So, what is a Hack Architecture anyway?

Hack Architecture is a simple computer based on Von-Neuman Architecture that enables implementation of simple enough instruction sets to function as a general purpose computer. The Hack computer is based on Harvard Architecture where the instruction and the data memory are separate, and registers in CPU access them as and when required. The Hack Computer is fundamentally built on NAND and DFF gates and boasts of a separate data and instruction memory, both addressable by an Address Register in the CPU. When the computer is turned on, the ROM loads the OS Kernel into the main memory and then the OS further readies the computer to take user input.

The Hack Assembly Language

Although in principle, one can always provide instruction sets to the computer in machine language, Assembly language tends to make the job easier by abstracting the various aspects of machine instructions with human understandable symbols. Hack Assembly Language does the same for Hack Computers by encoding the information in terms of symbols and allowing user to use variables to represent the address memory. Broadly these are the few things to keep in mind about the Hack Assembly Language-

  • In Hack Architecture, the address register selects both the ROM and RAM adress. So in essence, the current queried address depends on use case. Refer here for example
  • The Hack Assembly Language feature three different types of instructions shown in the table. In addition, goto instruction simply directs the computer to associate the symbol with the succeeding line number without any actual machine instruction. Refer here for example
  • The Hack Assembly Language provides functionality to declare variables for ease of use. The user declared variable address starts at 16 and increments further as we define more variables. In addition, there are predefined symbols as well that refer to a specific pre-defined address. Refer here

How is HackML developed?

HackML is completely written in C++. On a high level, it functions as given below-

  • In the first go, the program reads the user input given in Hack Assembly Language and parses the variables in the first go. If there is any error in the way symbols are defined, the program terminates with an error
  • Assuming that all the symbols are defined properly, the symbols are recorded and the value stored in a symbol table. The internal symbols are initiated by this point.
  • In the second go, the computation instruction statements are parsed and matched with the computation universe of the Hack ALU. If at any stage, that is at destination, computation and jump, the instruction is found to be invalid, the program terminates throwing the corresponding error
  • Finally, as all the elements are parsed, the coder module provides binary code of all the parsed symbols and instructions.

Let's get you started on the tutorial to write programs in Hack Assembly Language!

Entering a constant

@17

Constants in the Hack Computer are always loaded using Address Registers. A is the symbol in the Hack Assembly Language for it.

Accessing Memory Content

// D=17 (D refers to the Data Register)
@17
D=A

// RAM[100]=17
@17
D=A
@100
// M refers to the memory register
M=D

// RAM[200] = RAM[100]
@100
D=M
@200
M=D

D stores the Data and M refers to the data memory.

Branching examples

// goto 29
@29
0;JMP

// if D>0 goto 63
@63
D;JGT

A register simultaneously accesses the Data and Instruction Memory together. What is used depends on the programming instruction as we saw in the earlier cases.

Variable use examples

// x = -1
@x
M=-1

// count = count-1
@count
M=M-1

//sum = sum + x
@sum
D=M
@x
D=D+M
@sum
M=D

Variables are declared by the users to abstract away the memory requirements in a program.
